# Security Policy | 安全政策

## Supported Versions | 支持的版本

We release patches for security vulnerabilities for the following versions:

我们为以下版本发布安全漏洞补丁：

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability | 报告漏洞

We take the security of mini-infer seriously. If you believe you have found a security vulnerability, please report it to us as described below.

我们非常重视mini-infer的安全性。如果你认为发现了安全漏洞，请按照以下说明向我们报告。

### How to Report | 如何报告

**Please do not report security vulnerabilities through public GitHub issues.**

**请不要通过公开的GitHub issue报告安全漏洞。**

Instead, please send an email to: **<smarter.shh1104@gmail.com>**

请发送电子邮件至：**<smarter.shh1104@gmail.com>**

Please include the following information in your report:

请在报告中包含以下信息：

- Type of vulnerability (e.g., buffer overflow, code injection, etc.)
- Full paths of source file(s) related to the vulnerability
- The location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

- 漏洞类型（例如缓冲区溢出、代码注入等）
- 与漏洞相关的源文件的完整路径
- 受影响源代码的位置（标签/分支/提交或直接URL）
- 重现问题的分步说明
- 概念验证或利用代码（如果可能）
- 问题的影响，包括攻击者如何利用它

### Response Timeline | 响应时间表

- We will acknowledge receipt of your vulnerability report within **3 business days**
- We will send a more detailed response within **7 business days** indicating the next steps
- We will keep you informed about the progress towards a fix and announcement
- We may ask for additional information or guidance

- 我们将在**3个工作日**内确认收到你的漏洞报告
- 我们将在**7个工作日**内发送更详细的响应，说明后续步骤
- 我们将及时告知你修复和公告的进展
- 我们可能会要求提供其他信息或指导

### Disclosure Policy | 披露政策

- We request that you give us reasonable time to fix the vulnerability before public disclosure
- We will publicly acknowledge your contribution (unless you prefer to remain anonymous)
- We will coordinate the timing of the public disclosure with you

- 我们要求你在公开披露之前给我们合理的时间来修复漏洞
- 我们将公开感谢你的贡献（除非你希望保持匿名）
- 我们将与你协调公开披露的时间

## Preferred Languages | 首选语言

We prefer all communications to be in English or Chinese (中文).

我们希望所有通信都使用英语或中文。

## Comments on this Policy | 对此政策的意见

If you have suggestions on how this process could be improved, please submit a pull request.

如果你对如何改进此流程有建议，请提交pull request。
